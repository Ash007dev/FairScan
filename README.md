# FairScan — AI Fairness Audit Dashboard

> Built for **Google AI Hackathon 2026** · Detects hidden bias in AI decision systems in under 30 seconds.

---

## What is FairScan?

FairScan is an AI-powered fairness auditing tool that analyzes any decision dataset and surfaces bias across demographic groups. Upload a CSV, select the decision column, and four specialized AI agents run in parallel to deliver a complete audit with legal compliance analysis, root cause attribution, a counterfactual demo, and a plain-English compliance memo.

---

## Features

| Feature | Description |
|---|---|
| **Statistical Analysis** | Approval rates per group, Adverse Impact Ratio (EEOC 4/5ths rule), demographic parity, equalized odds |
| **Root Cause (SHAP)** | Identifies which column drives the bias using SHAP feature importance |
| **Legal Mapper** | Flags violations of EU AI Act, EEOC, RBI guidelines via Gemini AI |
| **AI Report** | Plain-English compliance memo generated by Gemini 2.0 Flash, exported as PDF |
| **Counterfactual Engine** | Flip one demographic attribute and watch the model decision change in real-time |
| **Dynamic Column Support** | Works with any CSV — sensitive columns auto-detected, counterfactual buttons built from actual data |
| **Cross-Agent Validation** | 4 agents cross-check each other; inconsistencies are flagged and shown |

## Quick Start

### 1. Clone the repo
```bash
git clone https://github.com/Ash007dev/FairScan.git
cd FairScan
```

### 2. Backend setup
```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure API key
cp .env.example .env
# Edit .env and add your GEMINI_API_KEY

# Start the server
uvicorn main:app --reload
# ✓ Backend at http://localhost:8000
# ✓ API docs at http://localhost:8000/docs
```

### 3. Frontend setup
```bash
cd frontend
npm install

# .env.local already contains:
# NEXT_PUBLIC_API_URL=http://localhost:8000

npm run dev
# ✓ Frontend at http://localhost:3000
```

### 4. Run your first audit
1. Open `http://localhost:3000`
2. Upload any CSV with a binary decision column — or click **"Load UCI Adult Income demo"** for a built-in example with known gender bias
3. Select the decision column (highlighted in **cyan**)
4. Sensitive columns (age, sex, race, etc.) are auto-detected and shown in **purple**
5. Click **"Run bias audit"** and watch 4 agents run in real-time

## UI Overview

### Upload Page
- Drag-and-drop or click to upload a CSV
- Columns auto-parsed from the first row
- Sensitive columns highlighted in **purple**, selected decision column in **cyan**
- One-click demo dataset loader

- Gradient progress bar

- **Fixed left sidebar** with 7 navigable sections — no more endless scrolling

## Known Limitations

| Item | Status | Notes |
|---|---|---|
| Gemini free-tier quota | ⚠ Active | Free tier has daily/minute limits. App falls back to f-string memo if quota is exceeded. |
| `pickle.loads()` in counterfactual | ⚠ Known risk | Model is stored server-side and round-tripped through the client. Acceptable for hackathon demo; should be keyed server-side for production. |
| CORS `allow_origins=["*"]` | ⚠ Dev only | Lock to Firebase domain before any real deployment. |
| In-memory audit store | ✅ Mitigated | Capped at 50 entries with LRU eviction (`CappedAuditStore`). |
